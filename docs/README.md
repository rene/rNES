## Architecture

Fig. 1 shows the rNES architecture. There are three layers represented by dotted lines.
Bottom layer is the core of emulator, which represents the NES hardware and where almost everything
happens. The top layer represents the interface with real world, such as video generator, input
controllers and sound output. This separation is done to allow portability over different scenarios.
For instance, input controllers could be keys read from a physical keyboard or commands read from a
TCP connection.  Video output could be a desktop window or even a composite video (or VGA) signal
sent to a television or monitor. In both cases, the data generated by the emulator is the same, the
differences are only at the top layer implementation. Both layers are connect through a middle layer
that provides a common API to connect the emulator and physical interfaces (video, audio, etc).

User interface can access all layers at any level, i.e., call functions provided by each module
to control the whole emulation, providing graphical interfaces for configuration management or
debug.

![rNES Architecture](/docs/imgs/rNES_Arch.png)

*Fig. 1: rNES architecture*

The core of rNES is composed mainly by the following modules:

*  **System Bus**: This is basically a *Memory Management Module*. Since on
   NES systems all peripherals and internal hardware are memory mapped,
   this module is responsible for control memory access and provide
   mechanisms to execute the memory operations of the corresponding modules
   according to the input address. For instance, when address 0x2002 is
   read, *System Bus* should call the *PPU* module to read PPU status and
   return correct value at that moment. Note that modules which has
   internal memory (such as the VRAM from PPU) can also handle their memory
   internally.

*  **CPU**: This module implements the 6502 processor presented on NES
   systems. It works executing the Von Neumann Cycle: fetching data from
   main memory, decoding and executing instructions. It also provides
   functions to trigger CPU interrupts.

*  **ROM Decoder**: Responsible for load game ROMs into main memory. Since
   several games has on their cartridge additional hardware (*mappers*),
   this module should provide all necessary information about the game and
   inform their *mappers* to *Cartridge* module. All *mappers* of the
   game needs to be implemented by the emulator, otherwise the game will
   not work correctly.

*  **Cartridge Mappers**: This module implements many different
   *mappers* used by the games and the behavior of the game
   cartridges itself. It should map on the System Bus the PRG and CHR
   memory areas and, depending on the mapper needed, control memory banks,
   PPU mirroring, etc. Thus, the PPU's VRAM is also mapped by this module.

*  **Controller**: This module handles the game controller ports,
   implementing the memory mechanism for reading buttons.

*  **APU**: This module implements the Audio Processing Unit, which is
   responsible for audio generation. Note that APU should implement only
   the five channels present on the real hardware (two pulse wave
   generators, a triangle wave, noise, and delta modulation channel). Only
   signals are generated, audio output is done by top layer.

*  **PPU**: This module implements the Picture Processing Unit, which is
   responsible for video handling. This module should care about timing
   constraints, providing features to allow PAL and NTSC games. The video
   generation itself (since draw pixels or generate a composite video
   signal) is done by top layer.

*  **Clock Generator**: This module provides clock signals at the correct
   (and desirable) time for APU, CPU and PPU. This module is not mandatory
   for all platforms, the synchronization of the emulation can be achieved
   by syncing the CPU/PPU clock cycles with the audio sampling that is done
   on a very precise rate, e.g. 44100 Hz.

*  **System Palette**: This module provides RGB colors representing the
   system color palette. Colors shall be customizable.

## Source Layout

```
src/
├── apu.c                 ← APU module
├── cartridge.c           ← Cartridge module (ROM loading and mapper dispatch)
├── controller.c          ← Joypad register emulation
├── cpu/                  ← CPU module (6502 emulation)
├── gui
│   └── sdl/              ← Reference HAL implementation (SDL2)
│       ├── audio.c       ← SDL2 audio device
│       ├── clock.c       ← SDL2 timer
│       └── gui.c         ← Window, input, pixel buffer
├── hal
│   └── fileops.c         ← Platform file I/O (load_file)
├── include/              ← Header files
├── interface
│   └── posix
│       └── rNES.c        ← rNES entry point (posix implementation)
├── mappers/              ← Mapper implementations (0 to 7)
├── palette.c             ← Default NES palette
├── ppu.c                 ← PPU module
├── ringbuffer.c          ← Ring buffer implementation
├── romdec.c              ← ROM decoder module (iNES / NES 2.0 formats supported)
└── sbus.c                ← System bus + DMA
```
