# rNES Porting Guide

This document is a complete guide for porting rNES to a new platform. All
information is derived from the Doxygen documentation generated by `make docs`.

rNES is designed with portability as a first-class goal. The emulator core вҖ”
CPU, PPU, APU, system bus, cartridge, and controller вҖ” is fully platform-independent
C code. All platform-specific concerns are isolated behind a **Hardware Abstraction
Layer (HAL)** declared in `src/include/hal/hal.h`.

and `src/interface/<myplatform>/main.c` for your entry point.

## Thread Model Summary

The reference port uses two threads:

- **GUI thread** вҖ” calls `gui_init()`, then blocks in `gui_main_loop()`.  
- **Emulator thread** вҖ” loops on `sbus_clock()` until `quit_emulation` is set.

Emulation speed is governed entirely by audio backpressure: the APU ring buffer
fills up and the emulator thread stalls until the audio callback drains it at
44100 Hz. See [Thread Model](thread-model.md) for the full data-flow diagram.

### File I/O


## Emulator Module Init/Destroy Order

The correct initialization order is mandatory. This mirrors `rNES.c` exactly.

### Initialization

```c
/* 1. Initialize system bus */
sbus_init();

/* 2. Load and attach cartridge */
cartridge_load(romfile, &cartridge);
sbus_set_cartridge(cartridge);

/* 3. Initialize subsystems */
cpu_init();
ppu_init();
apu_init();

/* 4. Register callbacks (before starting emulator thread!) */
cpu_register_kill_cb(my_kill_cb);   /* optional: handle CPU kill */
cpu_register_debug_cb(my_debug_cb); /* optional: very slow */
sbus_register_debug_cb(my_sbus_cb); /* optional: very slow */

/* 5. Reset all subsystems */
controller_reset();
ppu_set_cartridge(cartridge);
cpu_reset();
ppu_reset();
apu_reset();

/* 6. Start threads (GUI first, then emulator) */
```

> **Thread safety warning:** `cpu_register_kill_cb`,
> `cpu_register_debug_cb`, and `sbus_register_debug_cb` are **not thread safe**.
> Always register callbacks **before** starting the emulator thread.

### Destruction

```c
ppu_stop();           /* disable rendering before freeing PPU */
ppu_destroy();
cpu_destroy();
apu_destroy();        /* closes ring buffer, unblocks any waiting rbuff_put */
sbus_destroy();
cartridge_unload(cartridge);
gui_destroy();        /* close window and audio device */
```

## Step-by-Step Porting Checklist

### 1. Create platform directories

```
src/gui/<myplatform>/
src/interface/<myplatform>/
```

### 2. Implement `load_file`

Load the ROM file into a *malloc-allocated* buffer. On embedded targets, read
from flash or a filesystem abstraction. Return the byte count or a negative
error code.

### 3. Implement `gui_init` / `gui_destroy`

- Open a framebuffer or window of `256*scale Г— 240*scale` pixels.
- Inside `gui_init`, call `audio_init()` and `audio_play()`.
- Return `0` on success, `-1` on failure.

### 4. Implement the pixel buffer (`gui_set_pixel` / `gui_get_pixel`)

Maintain a local `RGB_t` array of size `256 Г— 240`. `gui_set_pixel` writes into
it; presentation (scaling and blitting to the hardware display) happens in
`gui_main_loop`.

### 5. Implement `gui_read_joypad`

Map physical buttons or keyboard keys to the NES bitmask:

| Bit | Button |
|-----|--------|
| 7 | A |
| 6 | B |
| 5 | Select |
| 4 | Start |
| 3 | Up |
| 2 | Down |
| 1 | Left |
| 0 | Right |

This function is called from the **emulator thread**. Protect shared input state
with atomics or a mutex if it is updated in the GUI thread.

### 6. Implement `gui_main_loop`

- Poll or block on platform events.
- Present the pixel buffer to the display once per frame (~16.7 ms for NTSC).
- Return when the user requests quit.

### 7. Implement `audio_init` / `audio_destroy`

Open an audio device at `AUDIO_SAMPLE_RATE` (44100) Hz, mono, signed 16-bit
samples. Do not start playback yet.

### 8. Implement `audio_set_cb`

Store the provided `audio_cb` and `data` pointer. Arrange for the audio device
driver to call `audio_cb(data, stream, len)` each time it needs a new buffer.
This is the primary emulation clock вҖ” it must fire at 44100 / 2048 вүҲ 21.5 Hz.

### 9. Implement `audio_stream_put`

Copy `len` bytes of audio samples from `data` into `stream` with `volume` scaling
(range 0вҖ“128). This is called from inside the audio callback.

### 10. Implement `audio_play` / `audio_stop`

Start and pause the audio device respectively.

### 11. Define the five color constants

In your GUI implementation file:
```c
const RGB_t C_BLACK = {0,   0,   0  };
const RGB_t C_WHITE = {255, 255, 255};
const RGB_t C_RED   = {255, 0,   0  };
const RGB_t C_GREEN = {0,   255, 0  };
const RGB_t C_BLUE  = {0,   0,   255};
```

### 12. Write the entry point

Follow the module init order from the previous section. Use a semaphore or
equivalent to ensure the emulator thread starts only after `gui_init()` returns.

### 13. Start the two threads

- Start the **GUI thread** first. It calls `gui_init()`, then posts to the
  semaphore, then blocks in `gui_main_loop()`.
- Wait on the semaphore.
- Start the **emulator thread**: `while (!quit_emulation) sbus_clock();`
- Join the GUI thread on exit, then destroy all modules.

### 14. Build and test

Start with a Mapper 0 ROM (NROM, e.g. Donkey Kong, Super Mario Bros.) before
testing mappers 1вҖ“7.
